# 🚀 Real-time AI Dashboard - Complete Setup Guide\n\n**AI-powered task management that monitors Gmail, Notion, Slack, and Fireflies in real-time.**\n\n## ⚡ Quick Start (5 minutes)\n\n```bash\n# 1. Clone and setup backend\ngit clone https://github.com/grizzcodes/realtime-ai-dashboard.git\ncd realtime-ai-dashboard/backend\nnpm install\ncp .env.example .env\n\n# 2. Setup frontend\ncd ../frontend\nnpm install\n\n# 3. Configure environment (see below)\n# 4. Start the servers\n```\n\n## 🔧 Environment Setup\n\n### Required Services\n\n1. **Supabase Database** (Required for persistence)\n   - Go to [supabase.com](https://supabase.com)\n   - Create new project: `realtime-ai-dashboard`\n   - Run SQL from `docs/supabase-setup.md`\n   - Add URL and key to `.env`\n\n2. **AI Services** (At least one required)\n   - **OpenAI**: Get key from [platform.openai.com](https://platform.openai.com/api-keys)\n   - **Claude**: Get key from [console.anthropic.com](https://console.anthropic.com/)\n\n3. **Optional Integrations**\n   - **Notion**: [notion.so/my-integrations](https://www.notion.so/my-integrations)\n   - **Slack**: [api.slack.com/apps](https://api.slack.com/apps)\n   - **Google**: [console.cloud.google.com](https://console.cloud.google.com/)\n\n### Environment Variables\n\nEdit `backend/.env`:\n\n```bash\n# Required\nSUPABASE_URL=your_supabase_project_url\nSUPABASE_ANON_KEY=your_supabase_anon_key\nOPENAI_API_KEY=your_openai_key  # OR\nANTHROPIC_API_KEY=your_claude_key  # OR both\n\n# Optional\nNOTION_API_KEY=your_notion_integration_token\nSLACK_BOT_TOKEN=xoxb-your-slack-bot-token\nGOOGLE_CLIENT_ID=your_google_oauth_client_id\nGOOGLE_CLIENT_SECRET=your_google_oauth_secret\n```\n\n## 🏃‍♂️ Running the Application\n\n### Development Mode\n\n```bash\n# Terminal 1: Start backend\ncd backend\nnpm run dev\n\n# Terminal 2: Start frontend\ncd frontend\nnpm start\n```\n\n### Production Mode\n\n```bash\n# Backend\ncd backend\nnpm start\n\n# Frontend (build and serve)\ncd frontend\nnpm run build\n# Serve with your preferred method\n```\n\n## 🎯 Features Overview\n\n### ✅ Current Features\n- **Real-time AI Analysis**: OpenAI/Claude processing of events\n- **Multi-source Integration**: Slack, Gmail, Notion, Fireflies\n- **Intelligent Task Creation**: AI automatically creates actionable tasks\n- **Live Dashboard**: Real-time updates via WebSockets\n- **Priority Management**: Urgent tasks highlighted automatically\n- **Database Persistence**: All tasks and events stored in Supabase\n- **Responsive UI**: Works on desktop and mobile\n\n### 🔄 Real-time Updates\n- New tasks appear instantly\n- Status changes broadcast to all clients\n- Live activity feed\n- Automatic stats updates\n\n## 📱 Using the Dashboard\n\n### Task Management\n1. **View Tasks**: Organized by urgency (Critical → Low)\n2. **Complete Tasks**: Click checkmark to mark done\n3. **AI Insights**: See AI confidence and analysis\n4. **Filters**: Pending vs completed tasks\n\n### Testing AI\n1. Click \"Test AI\" button in dashboard\n2. AI will analyze a sample message\n3. Watch new tasks appear in real-time\n4. Check activity feed for event processing\n\n### Webhook Integration\n- **Slack**: Set webhook URL to `http://your-domain:3001/webhooks/slack`\n- **Gmail**: Configure Pub/Sub to `http://your-domain:3001/webhooks/gmail`\n- **Notion**: Add webhook URL in database settings\n\n## 🛠 Development\n\n### Backend Structure\n```\nbackend/\n├── main.js                    # Main server with WebSocket\n├── src/\n│   ├── ai/intelligentProcessor.js    # AI event processing\n│   ├── database/supabaseClient.js    # Database integration\n│   ├── services/notionService.js     # Notion API client\n│   └── webhooks/webhookHandler.js    # Multi-service webhooks\n└── scripts/test-webhooks.js          # Testing utilities\n```\n\n### Frontend Structure\n```\nfrontend/src/\n├── App.js                     # Main dashboard app\n├── components/\n│   ├── TaskBoard.js          # Task management UI\n│   ├── ActivityFeed.js       # Real-time event feed\n│   └── StatsPanel.js         # Dashboard statistics\n└── App.css                   # Tailwind + custom styles\n```\n\n### API Endpoints\n- `GET /` - Dashboard info and stats\n- `GET /api/tasks` - Get all tasks\n- `PUT /api/tasks/:id/status` - Update task status\n- `GET /api/events` - Recent events\n- `POST /api/ai-test` - Test AI processing\n- `GET /api/notion/*` - Notion integration\n\n## 🚨 Troubleshooting\n\n### Common Issues\n\n**\"AI processing failed\"**\n- Check API keys in `.env`\n- Verify OpenAI/Claude credits\n- Check network connectivity\n\n**\"Supabase not configured\"**\n- Verify `SUPABASE_URL` and `SUPABASE_ANON_KEY`\n- Run database schema from `docs/supabase-setup.md`\n- Check Supabase project is active\n\n**\"Real-time updates not working\"**\n- Check WebSocket connection in browser dev tools\n- Verify frontend connects to correct backend port\n- Check firewall/proxy settings\n\n**\"Webhooks not receiving data\"**\n- Use ngrok for local development: `ngrok http 3001`\n- Check webhook URLs in service settings\n- Verify webhook authentication\n\n### Debug Mode\n\n```bash\n# Enable detailed logging\nLOG_LEVEL=debug npm run dev\n\n# Test webhook processing\nnpm run test\n```\n\n## 🎯 Next Steps\n\nAfter basic setup:\n\n1. **Configure Real Webhooks**: Set up actual service integrations\n2. **Customize AI Prompts**: Modify analysis logic in `intelligentProcessor.js`\n3. **Add Authentication**: Implement user accounts and permissions\n4. **Deploy**: Use Docker or cloud platforms\n5. **Scale**: Add Redis for multi-instance support\n\n## 📊 Monitoring\n\n- **Health Check**: `GET /health`\n- **Database Stats**: View in Supabase dashboard\n- **Real-time Metrics**: Built into dashboard UI\n- **Logs**: Check console output for processing details\n\n## 🤝 Contributing\n\n1.
